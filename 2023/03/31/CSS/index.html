position 定位

position属性指定了元素的定位类型，不同的定位方法有不同的工作方式

  1. static定位
    position的默认值，即没有定位，遵循正常的文档流对象。静态定位的元素不会受到top、bottom、left、right的影响
  2. fixed定位
    元素的位置相对于浏览器窗口是固定位置，即使窗口是滚动的它也不会移动。Fixed定位使元素的位置与文档流无关，因此不占据空间。Fixed定位的元素和其他元素重叠。
  3. relative定位
    相对于元素原来的位置进行定位，不会释放出原本的空间
  4. absolute定位
    绝对定位的元素的位置相对于最近的已定位父元素，如果元素没有已定位的父元素，那么它的位置相对于<html>，一般是子绝父相。会脱离标准文档流
  5. sticky定位

Grid网格布局

flex布局为一维布局，grid布局为二维布局

容器和项目

采用网格布局的区域称为容器，即上面例子里最外层的div。
容器内部采用网格定位的子元素，称为项目。内层的三个div就是项目。
⚠️ 项目只能是容器的顶层子元素，不包含项目的子元素。如以上的<p>元素就不是项目。grid布局只对项目生效

行、列、单元格、网格线

水平的深色区域就是行，垂直的深色区域就是列
行和列的交叉区域为单元格cell，n行和m列会产生nxm个单元格
划分网格的线-网格线，n行有n+1根水平网格线，m列有m+1根垂直网格线

布局的属性分为：容器属性、项目属性

容器属性：定义在容器上

默认情况下，容器元素都是块级元素，但也可以设为行内元素

⚠️设为网格布局以后，容器子元素（项目）的float、display: inline-block、display: table-cell、vertical-align和column-*等设置都将失效。
容器设定了网格布局后，就要划分行和列。grid-template-columns属性定义每一列的列宽，grid-template-rows属性定义每一行的行高。

grid-row-gap属性设置行与行的间隔（行间距），grid-column-gap属性设置列与列的间隔（列间距）。
