产品经理

用户篇

对用户的认知处于什么水平？

  1. 你能描述产品的核心用户画像吗？（考察对用户的熟悉程度以及如何去形容用户）
  2. 你能描述用户使用产品的典型行为吗？（考察用户在app上行为描述，用定量的语句描述）
  3. 你知道产品用户的分群逻辑是什么吗？（考察对用户细分的认知和细分的逻辑）
  4. 你知道不同用户群体的需求和运营方式？（考察对用户分群的理解和运营）

那如果能够清晰回答以上问题，说明具备了

  1. 基础用户认识、能描述用户特征的能力
  2. 使用数据模型去定义、概括用户的能力
  3. 细分的思维，挖掘特征进行分层的能力
  4. 对用户有深入理解，做分类运营的能力

用户感

用户感：收集、分析、预测用户行为的能力
收集：理性地去调研、收集用户信息
分析：去伪存真分析用户实际需求

用户调研

  • 从具象到具象 ：男性用户有多少人？
  • 从具象到抽象 ：大学生用户对这款产品的看法是什么？
  • 从抽象到具象 ：性格内向的人有多少人？
  • 从抽象到抽象 ：社交达人如何评价我们的产品？

底层逻辑
定性：非数据分析得结论，为什么、怎么样
定量：数学分析数据有多少、有多大
行为：用户做了啥，数据记录实际操作
态度：用户说了啥，用户自己来表达

不同场景适用的调研方法

  1. 定量+行为 = 数据说用户做了啥
  2. 定量+态度 = 用户书面反馈
  3. 定性+行为 = 看用户做了啥
  4. 定性+态度 = 和用户沟通

用户体系

用户体系是用画像、标签去构建分群，根据每个分群制定策略

  1. 用户画像：构建一个用户的虚拟体，达成对用户的共识（讲故事）
  2. 用户标签：以标签来描述用户，形成具象行为的用户（贴标签）
  3. 用户分群：根据特征进行分层，适合分类运营

例子：

用户画像

  1. Persona 虚拟用户画像：由感性词构成一个故事。属于比较抽象的、感性的画像。用于打造出对用户认知的共识；用于了解用户、匹配产品等宏观场景
  2. Profile 数据用户画像：由一系列标签构成一个档案。属于比较理性、偏标签类的画像。用于标记一个、一群用户的详细信息；用于用户运营、营销等实用场景

从具象到抽象的步骤（persona）

需求背景 -> 个体分析 -> 关键特征 -> 丰富细节

  1. 个体分析：关键是找到典型用户，能最大限度去代表所研究的用户
    符合：
  • （1）产品的目标或使用用户
  • （2）所研究“需求”或“问题”的直接用户
  • （3）能代表所研究的群体
  • （4）是所研究的群体的均值，而非最值或极值
    方法：
  • 用户访谈
  • 调研问卷
  • 产品行为
  • 用户的关键特征分为：
    • 基础信息：是否有相似的背景？
    • 行为信息：是否有相似的行为方式？
    • 心理动机：是否有相似的内心思想？

具象到抽象的例子：如何增加美团骑手数量，

  1. 调研个体，找到典型用户：找到收入处于中位数的骑士，了解他们的工作现状和收入情况；找到刚加入的骑手，了解他们加入的渠道、动机和原因
  2. 找到关键特征：分析他们的基础信息、行为信息、心理动机
  • 基础信息：农村出身为主、90后、男性为主、大专以下、原收入低于5k
  • 行为信息：好奇心强、熟悉手机操作、喜欢打王者、每日配送35单左右、容易月光
  • 心理动机：不想太复杂的工作、不太喜欢坐班、希望时间自由一些、喜欢有朋友的氛围
  1. 丰富细节，建立有灵魂的画像
  2. 验证和应用
  • 用户获取
    • 让骑手有偿推荐、普通劳动力市场、各地的外包公司->是否有更多精准用户？
  • 用户运营
    • 电车租赁、后付费、工资结算时间快、包吃住->是否更容易转化骑手？
  • 更多需求
    • 相亲活动、社群活动、技能提升->是否让骑手更容易留存？

从抽象到具象：如何将抽象概念落地应用（profile）

  1. 抽象集合persona
  2. 数据标签：数据采集、定义标签、给用户打标签
  3. 用户分群
  4. 场景应用

完整用户画像的构建和使用流程

遇到实际情况 -> 从问题中提炼用户画像 -> 从用户画像中挖掘具体数据 -> 实际应用场景

用户标签

用户画像是对外展示的有血有肉的一面，用户标签是以数据、逻辑支撑起整套体系
用户标签是用户某一维度特征的属性描述，分为

  • 静态标签：容易获得，可以作为辅助标签
  • 动态标签：随着用户与平台持续交互，数据不断更新
